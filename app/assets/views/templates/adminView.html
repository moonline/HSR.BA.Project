Manage Request Templates

Saving ... Saved successfully! Could not save! Please try again, reload the page or file a bug report.
EEPPI works with different Project Planning Tools and Project Planning Tool configurations. However, it needs a template to know how to create requests to generate a task in your Project Planning Tool. You can define such Request Templates here. All Project Planning Tools are supported, if they can create tasks over a web request with HTTP basic authentication and a JSON request. For example Jira and Redmine are supported. In the Request Body Template you should provide a Json to perform the request to the PPT. In Request Body Template you can:
  • include variables with ${Variable name}. "taskTemplate", "node", "pptProject" and in post Processors also "parentRequestData" can be used. You can access their properties with the "."-operator (e.g. ${taskTemplate.name}).
  • call Processors with $Processor name:(parameter, another parameter, ...)$. Parameters can be variables and/or "strings". Strings may contain anything other than commas (,) including any quote characters (' " « »).
  • call post Processors almost like Processors with $!Processor name:(parameter, another parameter, ...)$ (note the exclamation mark). However, they are only processed after the parent Tasks request was executed. The result of the parent Tasks request can then be accessed with the variable "parentRequestData".
